RTS-style AI using Goal-Oriented Action Planner (GOAP)

Heat Visualization

3x Simulation Speed

For my specialisation, I developed an RTS-style AI inspired by Age of Empires. The core system revolves around a base controlled by a custom Goal-Oriented Action Planning (GOAP) system, supported by a heuristic-driven ActionPlanner. This planner evaluates potential actions up to a fixed depth, ignoring paths that exceed the estimated cost to a goal state.

The base uses world state data—resources, unit counts, and population cap—to dynamically determine optimal action sequences. Each action is weighted based on contextual value, allowing the system to prioritize high-impact decisions with lower heuristic costs.

Idle farmer units receive these plans and carry them out using A*-influenced steering. Rather than full pathfinding, they evaluate available grid steps based on distance to target and enemy heat influence.

Soldiers use a similar method, gravitating toward areas of high enemy presence without overlapping allied units. In the absence of enemies, they shift toward the edges of farmer influence, creating an emergent patrol behavior.

Goblins exhibit randomized wandering by selecting steps from a shuffled list and avoiding previously visited tiles. They prefer tiles with low soldier presence, or alternatively, high farmer influence—resulting in dynamic, unpredictable movement patterns.

The game world is grid-based, with procedurally placed resources and a centrally spawned base. This procedural layout ensures varied outcomes in each simulation run.

The architecture uses a simple entity system where all actors and objects inherit from a common base class. Rendering is optimized via sprite batching, and UI elements are included to provide visual clarity and feedback.
